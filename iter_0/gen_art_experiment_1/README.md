## Layman Summary

Tests whether limiting parameter change per EM step (KL-BW) improves HMM decoding when the model order is misspecified.

## Full Summary

Implements a reproducible HMM order-misspecification experiment comparing Standard BW, Warm-Start BW, and KL-Constrained BW across 1,260 results (315 main configs + 105 ε-sensitivity, all validated).

**Data generation**: For each (seed, k_true, d, T), a k_true-order HMM is simulated with A_true (d^k_true × d, diagonal-boosted for spectral gap ≈ 0.35) and B_true (d × d). Observations and hidden states produced by forward simulation.

**Methods**:
- **BW** (baseline): First-order HMM, 3 restarts × 50 EM iterations, best log-likelihood chosen. CAR=0.0.
- **BW-warm** (baseline): Continues BW best restart for 30 more unconstrained EM steps. CAR=0.0.
- **KL-BW** (ours): Initialises from BW best restart, runs 30 constrained EM iterations. Each M-step projects A_unc and B_unc via geometric interpolation: α ∈ [0,1] found by binary search to satisfy KL(A∥A_ref) ≤ ε/2 and KL(B∥B_ref) ≤ ε/2. CAR = fraction of iters where constraint was binding (α < 1).
- **KL-BW-sensitivity** (ours, d=16 only): KL-BW at ε ∈ {0.06, 0.36, 0.5}.

**Evaluation**: Viterbi decoding with best-permutation accuracy (Hungarian matching) to resolve label swapping.

**Grid**: 5 seeds × 3 k_true × 3 d × 7 T = 315 main configs; 9 parallel workers; 12-min runtime.

**Validation passed**: 945 main + 315 sensitivity results; all viterbi_acc ∈ [0,1]; BW/BW-warm CAR=0; schema validated against exp_gen_sol_out.
